CSS (Style Sheets)

Estructure

selectors: div, li...

declarations: color: red; font-size: bold;...

Estructure: selector {declaration ; declaration;}

Types of CSS

  1. CSS inside HTML page: style tag inside the head and all the selectors and declarations inside those tags.
  2. External CSS file: new file with .css extension, ie. styles.css and we link that file to the HTML pages. ML pages. We use t We use the link tag in the HTML page with the relative path to that HTML pages. The structure of that tag is: link rel="stylesheet" href="css/styles.css"

Types of declarations

  • color: brown;
  • font-size: 30px;
  • background-color: gold;
  • text-decoration: underline; (subrallado) o line-through (tachado) o none (para eliminar caracteristicas por defecto ej. linea de los enlaces)
  • font-family: Arial, Helvetica, sans-serif; (It should hold several font names because if the browser does not support the first font, it tries the next font. For more info: W3schools CSS Web Safe Font Combinations )
  • text-align: center / right / left;
  • column-count: 3; (To divide the text in 3 columns)
  • column-gap: 40px;
  • border: 4px red solid; (to add borders. It includes border-width, border-style and border-color
  • /* */ : to add comments.
  • list-style-type: square / none; (to have square list symbols instead of the deffault discs or hide them)
  • text-shadow: 2px 5px grey; (horizontal vertical colour)
TIP: It's important to use Web safe fonts to make sure that all users can see the same font in their browsers.

Inline elements & Block-level elements

  • Inline elements: only use the space that they need ie. span, em, strong, a...
  • Block-level elements: each one starts in a new line ie. div, p, h1 to h6, ul, li...
TIPS: We can transform an inline element into a block-level element and viceversa with the CSS display declarator. The inline-block property has the best of both worlds for inline elements (inline element with the margin if block-level elements.
Margin (space outsidehttps://www.w3schools.com/cssref/css_websafe_fonts.asp a box or border) only add space on left and right hand-side for inline elements. It also add spaces on top and bottom for block-level elements.
Padding (space inside the border) works the same way for both. It surounds all the element.

CSS Classes & Selectors

If we add the attribute class to any HTML tag we can style it with CSS. The syntax in CSS is .classname{}. We can add the = class to different types of elements. Therefore if we only want to apply it ot 1 type we type: strong.purple. We can also 2 two classes to a selector and the systax in CSS is p.purple.green

IDs: We may use id instead of class to hook elements but since they are unique for 1 element, it's better to use the class attibute. We will need id attibutes for JavaScript. If we want to use id in HTML, the systax is p id="background" and it is #background or p#background in CSS.

Descendants

We can add style to the descendats of a element (elements inside that element) in CSS. The systaxis is:

  • div p: to style all the p tags inside div elements.
  • div .purple: to style the elements with the purple class inside divs.

Attribute selectors

a[href]{}: to style the a tags with that attribute.

a[href="https://www.w3schools.com/cssref/css_websafe_fonts.asp"]{}: to style only those links.

a[href*="colors"]{}: to style all the a tags with the word colors in its URL.

a[href$=".asp"]{}: to style all the a tags whose URL ends with .asp. It's useful for pdf files.

The Cascade

Inheritance: HTML elements can inherit some CSS properties that are applied to their parents. Fonts, colors... are inherited, but borders, margins, paddings...But we can inherit those values if we add the value inherit to those attributes in the children.

Specificity: we can overrule a parent rule but adding a different value in a child. div p is more specific than p.

More info: "Introducing the CSS Cascade" article.
